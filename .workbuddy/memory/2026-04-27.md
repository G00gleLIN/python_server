# 2026-04-27 工作日志

## 块块积木 CMD 转换功能实现完成

### 新增文件
1. **`cmd_protocol.py`** - 协议常量定义文件
   - 帧格式常量 (0x5B/0x5D, 每条CMD固定24字节)
   - type_main 枚举 (TM_VALUE/TM_TASK/TM_PROGRAM/TM_HARDWARE)
   - VAL_TYPE / CONS_TYPE 变量/常量类型枚举
   - param_set 位域工具函数 (build_param_set / parse_param_set)
   - ind 联合体工具 (build_ind / parse_ind)
   - **OPCODE_TO_CMD**: Scratch opcode → (type_main, type_sub) 映射表（核心！）
     - 数学运算(7)、比较(3)、逻辑(3)、字符串(4)
     - 控制(9)、变量(2)、列表操作(10)
     - SmartPi硬件扩展(18)
   - **MATHOP_MAP**: operator_mathop 运算符→type_sub 动态映射
   - **INPUT_NAME_MAP**: Scratch 输入参数名→协议参数位置(val1/val2/val3)映射
   - CRC8-Maxim 完整256条目查表 + crc8_maxim() 函数
   - build_frame() / parse_frame() 帧封装/解封函数

2. **`cmd_builder.py`** - CMD命令构建器
   - CmdBuilder 类: 积木块列表 → 24字节CMD命令列表 → 协议帧
   - _build_single_cmd(): 单个积木→24字节CMD
   - _parse_params(): 解析输入参数，生成 param_set + buff
   - _assemble_24byte(): 按 TM_VALUE/TM_PROGRAM 两种布局组装24字节
   - format_cmd_hex() / decode_cmd_detail(): 格式化/解码辅助

3. **`test_cmd_e2e.py`** - 端到端测试脚本

### 修改的文件
1. **`block_parser.py`**
   - 新增 `cmd_data`, `variables_found`, `lists_found` 属性
   - 新增 `_collect_cmd_data()` 方法: 收集结构化数据供CMD生成
   - 新增 `_extract_block_inputs()` 方法: 提取输入参数解析值
   - 在 `_execute_block`, `_execute_nested_block`, `_execute_condition_block`,
     `_execute_loop` 中调用收集CMD数据
   - `_parse_variables()` / `_parse_lists()` 现在同时写入 `variables_found`/`lists_found`

2. **`block_server.py`**
   - 导入 CmdBuilder, decode_cmd_detail
   - 新增 `generate_commands` action 路由
   - 新增 `generate_commands_data()` 方法: 解析项目→构建CMD帧→返回结果
   - 返回内容包含: cmd_stats, cmd_hex_strings, frame_hex, errors 等

### 测试结果 (Scratch作品3.sb3)
- 25个积木块 → 24条CMD命令 → 582字节协议帧
- 变量: 我的变量, X, Y (3个)
- 支持: 控制类(循环/if/wait), 运算类(+/-/*/比较/逻辑), 字符串,
       变量赋值, SmartPi硬件扩展(电机/LED/音符/红外)

### 设计要点
- type_sub 值以转换对照表Section 4为准（非C枚举），因C枚举值与实际协议不匹配
- 24字节CMD有两种内部布局:
  - TM_VALUE模式: [type_main][type_sub][ind][extend][len][buff16]
  - TM_PROGRAM模式: [type_main][type_sub][ind][param_set][buff18]
